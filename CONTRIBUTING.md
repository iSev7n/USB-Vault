# Contributing to USB Vault

Thank you for your interest in contributing.

USB Vault is a security-focused project. Stability and correctness are prioritized over rapid feature expansion.

---

## Code Style

- Follow existing formatting.
- Run before submitting:
  - npm run lint
  - npm run format
- Do not introduce unnecessary dependencies.
- Avoid heavy frameworks.

---

## Pull Requests

1. Fork the repository.
2. Create a feature branch.
3. Make focused changes.
4. Ensure builds pass:
   - npm ci
   - npm run dist
5. Submit a clear pull request description.

---

## Security Issues

If you discover a vulnerability:

- Do NOT open a public issue.
- Report privately following SECURITY.md.

---

## What Is Welcome

- Bug fixes
- Security improvements
- Documentation improvements
- UI refinements
- Cross-platform packaging improvements

---

## What Is Not Accepted

- Cloud integrations
- Tracking/analytics
- Telemetry
- Account systems
- Unnecessary dependencies

USB Vault is intentionally local-first and privacy-focused.
